# Security Policy

NASA Space Explorer is a small portfolio project, so security handling is simple and private.

## Reporting a Vulnerability

Do not post security vulnerabilities publicly in GitHub Issues.

Please report security concerns privately to the repository owner at [your-email@example.com](mailto:your-email@example.com).

When possible, include:

- A description of the issue
- Steps to reproduce it
- The potential impact
- Any suggested remediation

## Scope

This policy covers the project files and the public web app content used by the repository.

## Cryptography

This project does not implement custom cryptographic protocols or algorithms. By default, any network communication uses the browser's built-in HTTPS/TLS stack rather than project-defined cryptography.
